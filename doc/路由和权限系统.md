# 路由和权限系统技术文档

## 概述

本文档详细说明项目中实现的路由管理、懒加载、权限控制和 404 处理功能。

## 功能特性

### 1. 路由懒加载

所有页面组件都使用 `React.lazy()` 进行懒加载，减少初始加载时间，提升应用性能。

#### 实现方式

```typescript
// src/router/index.tsx
import { lazy } from 'react'
import { WithSuspense } from '@/components/WithSuspense'

// 懒加载页面组件
const Home = WithSuspense(lazy(() => import('@/page/home')))
const Dashboard = WithSuspense(lazy(() => import('@/page/dashboard')))
```

#### WithSuspense 高阶组件

`WithSuspense` 是一个高阶组件（HOC），用于包装懒加载组件并提供加载状态：

```typescript
// src/components/WithSuspense/index.tsx
export const WithSuspense = <P extends object>(
  Component: LazyExoticComponent<ComponentType<P>>
) => {
  const WrappedComponent = (props: P) => (
    <Suspense fallback={<LoadingFallback />}>
      <Component {...props} />
    </Suspense>
  )
  return WrappedComponent
}
```

**特点：**
- 自动包装 `Suspense` 组件
- 使用 `LoadingFallback` 显示加载动画
- 支持泛型，保留组件的 Props 类型
- 返回真正的组件函数，支持 React Hooks

### 2. 路由守卫（权限控制）

使用 `ProtectedRoute` 组件实现路由级别的权限控制。

#### 基础用法 - 需要登录

```typescript
{
  path: 'dashboard',
  element: (
    <ProtectedRoute>
      <Dashboard />
    </ProtectedRoute>
  ),
}
```

#### 高级用法 - 需要特定角色

```typescript
{
  path: 'admin',
  element: (
    <ProtectedRoute requiredRole="admin">
      <AdminPage />
    </ProtectedRoute>
  ),
}
```

#### ProtectedRoute 实现

```typescript
// src/components/ProtectedRoute/index.tsx
function ProtectedRoute({ children, requiredRole }: ProtectedRouteProps) {
  const { isAuthenticated, user } = useAuthStore()
  const location = useLocation()

  // 未登录，重定向到登录页
  if (!isAuthenticated) {
    return <Navigate to="/login" state={{ from: location }} replace />
  }

  // 需要特定角色权限
  if (requiredRole && user?.role !== requiredRole) {
    return <Navigate to="/403" replace />
  }

  return <>{children}</>
}
```

**工作流程：**
1. 检查用户是否已登录（`isAuthenticated`）
2. 如果未登录，重定向到 `/login`，并保存原始路径
3. 如果需要特定角色，检查 `user.role` 是否匹配
4. 权限不足时，重定向到 `/403`
5. 验证通过，渲染子组件

### 3. 认证系统

使用 Zustand 管理用户认证状态。

#### authStore 状态管理

```typescript
// src/store/authStore.ts
interface AuthState {
  isAuthenticated: boolean
  user: { name: string; role: string } | null
  login: (username: string, password: string) => Promise<boolean>
  logout: () => void
}

export const useAuthStore = create<AuthState>()(
  persist(
    set => ({
      isAuthenticated: false,
      user: null,
      login: async (username, password) => {
        // 模拟登录逻辑
        if (username === 'admin' && password === 'admin') {
          set({
            isAuthenticated: true,
            user: { name: 'Admin', role: 'admin' },
          })
          return true
        }
        return false
      },
      logout: () => {
        set({ isAuthenticated: false, user: null })
      },
    }),
    { name: 'auth-storage' }
  )
)
```

**特点：**
- 使用 `persist` 中间件持久化到 localStorage
- 支持登录/登出功能
- 存储用户信息和角色
- 刷新页面保持登录状态

#### 使用示例

```typescript
import { useAuthStore } from '@/store/authStore'

function LoginPage() {
  const { login } = useAuthStore()

  const handleLogin = async () => {
    const success = await login('admin', 'admin')
    if (success) {
      navigate('/dashboard')
    }
  }
}
```

### 4. 404 页面

自动处理未匹配的路由，显示友好的 404 页面。

#### 路由配置

```typescript

  path: '*',
  element: <NotFound />,
}
```

#### 404 页面实现

```typescript

function NotFound() {

  return (
    <Card>
      <h1>404</h1>

      <Button type="primary" onClick={() => navigate('/')}>

    </Card>
  )
}
```

## 技术亮点

### 1. 类型安全

所有组件和函数都使用 TypeScript 严格类型：

```typescript
// 泛型支持
export const WithSuspense = <P extends object>(
  Component: LazyExoticComponent<ComponentType<P>>
) => { ... }

// 接口定义
interface ProtectedRouteProps {
  children: React.ReactNode
  requiredRole?: string
}
```

### 2. 性能优化

- **代码分割**：懒加载减少初始包大小
- **按需加载**：只加载当前路由需要的代码
- **加载状态**：提供友好的加载动画

### 3. 用户体验

- **加载反馈**：使用 Ant Design Spin 组件
- **路径保存**：登录后自动跳转回原始页面
- **友好提示**：404 页面提供返回首页按钮

### 4. 可维护性

- **组件化**：功能拆分为独立组件
- **可复用**：`WithSuspense` 和 `ProtectedRoute` 可用于任何页面
- **易扩展**：添加新页面只需配置路由

## 注意事项

### 1. 登录逻辑

当前登录是模拟的，实际项目中需要调用后端 API：

```typescript
login: async (username: string, password: string) => {
  try {
    const response = await fetch('/api/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ username, password }),
    })
    const data = await response.json()
    
    if (data.success) {
      set({
        isAuthenticated: true,
        user: data.user,
      })
      return true
    }
    return false
  } catch (error) {
    console.error('登录失败', error)
    return false
  }
}
```

### 2. Token 管理

建议添加 token 管理和请求拦截器：

```typescript
interface AuthState {
  token: string | null
  setToken: (token: string) => void
}

// 请求拦截器
axios.interceptors.request.use(config => {
  const token = useAuthStore.getState().token
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### 3. 权限粒度

当前实现是路由级别的权限控制，如需更细粒度的权限控制（如按钮级别），可以创建权限判断 Hook：

```typescript
export const usePermission = (requiredRole?: string) => {
  const { user } = useAuthStore()
  return !requiredRole || user?.role === requiredRole
}

// 使用
function Dashboard() {
  const canDelete = usePermission('admin')
  
  return (
    <div>
      {canDelete && <Button>删除</Button>}
    </div>
  )
}
```

### 4. ESLint 配置

路由配置文件需要禁用 `react-refresh/only-export-components` 规则：

```typescript
/* eslint-disable react-refresh/only-export-components */
```

这是因为路由配置对象不是 React 组件，不需要 Fast Refresh。
