# 路由配置使用指南

## 功能说明

### 1. 懒加载（Lazy Loading）

所有页面组件都使用 `React.lazy()` 进行懒加载，减少初始加载时间。

```typescript
// 懒加载页面组件
const Home = lazy(() => import('@/page/home'))
const Dashboard = lazy(() => import('@/page/dashboard'))
```

**加载中状态：**
使用 Ant Design 的 `Spin` 组件显示加载动画。

### 2. 路由守卫（权限控制）

使用 `ProtectedRoute` 组件保护需要登录的路由。

#### 基础用法 - 需要登录

```typescript
{
  path: 'dashboard',
  element: (
    <ProtectedRoute>
      {withSuspense(Dashboard)}
    </ProtectedRoute>
  ),
}
```

#### 高级用法 - 需要特定角色

```typescript
{
  path: 'admin',
  element: (
    <ProtectedRoute requiredRole="admin">
      {withSuspense(AdminPage)}
    </ProtectedRoute>
  ),
}
```

### 3. 404 页面

访问不存在的路由时，自动显示 404 页面。

```typescript
{
  path: '*',
  element: withSuspense(NotFound),
}
```

## 认证系统

### authStore 状态管理

使用 Zustand 管理用户认证状态：

```typescript
import { useAuthStore } from '@/store/authStore'

// 获取认证状态
const { isAuthenticated, user, login, logout } = useAuthStore()

// 登录
const handleLogin = async () => {
  const success = await login('admin', 'admin')
  if (success) {
    console.log('登录成功')
  }
}

// 登出
const handleLogout = () => {
  logout()
}
```

### 认证状态持久化

认证状态会自动保存到 `localStorage`，刷新页面后保持登录状态。

### 示例 3：创建登录页面

```typescript
// src/page/login/index.tsx
import { useState } from 'react'
import { useNavigate, useLocation } from 'react-router-dom'
import { Form, Input, Button, Card, message } from 'antd'
import { useAuthStore } from '@/store/authStore'

function Login() {
  const navigate = useNavigate()
  const location = useLocation()
  const { login } = useAuthStore()
  const [loading, setLoading] = useState(false)

  const from = location.state?.from?.pathname || '/'

  const onFinish = async (values: { username: string; password: string }) => {
    setLoading(true)
    const success = await login(values.username, values.password)
    setLoading(false)

    if (success) {
      message.success('登录成功')
      navigate(from, { replace: true })
    } else {
      message.error('用户名或密码错误')
    }
  }

  return (
    <Card title="登录" style={{ maxWidth: 400, margin: '100px auto' }}>
      <Form onFinish={onFinish}>
        <Form.Item
          name="username"
          rules={[{ required: true, message: '请输入用户名' }]}
        >
          <Input placeholder="用户名: admin" />
        </Form.Item>
        <Form.Item
          name="password"
          rules={[{ required: true, message: '请输入密码' }]}
        >
          <Input.Password placeholder="密码: admin" />
        </Form.Item>
        <Form.Item>
          <Button type="primary" htmlType="submit" loading={loading} block>
            登录
          </Button>
        </Form.Item>
      </Form>
    </Card>
  )
}

export default Login
```

### 示例 4：在 Header 中显示登录状态

```typescript
// src/components/GlobalHeader/index.tsx
import { Button } from 'antd'
import { useNavigate } from 'react-router-dom'
import { useAuthStore } from '@/store/authStore'

function GlobalHeader() {
  const navigate = useNavigate()
  const { isAuthenticated, user, logout } = useAuthStore()

  const handleLogout = () => {
    logout()
    navigate('/login')
  }

  return (
    <Header>
      {isAuthenticated ? (
        <div>
          <span>欢迎，{user?.name}</span>
          <Button onClick={handleLogout}>退出登录</Button>
        </div>
      ) : (
        <Button onClick={() => navigate('/login')}>登录</Button>
      )}
    </Header>
  )
}
```

## 路由守卫工作流程

1. 用户访问受保护的路由（如 `/dashboard`）
2. `ProtectedRoute` 检查 `isAuthenticated` 状态
3. 如果未登录，重定向到 `/login`，并保存原始路径
4. 用户登录成功后，自动跳转回原始路径
5. 如果需要特定角色，检查 `user.role` 是否匹配

## 注意事项

### 1. 修改登录逻辑

当前登录是模拟的，实际项目中需要调用后端 API：

```typescript
// src/store/authStore.ts
login: async (username: string, password: string) => {
  try {
    // 调用后端 API
    const response = await fetch('/api/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ username, password }),
    })
    
    const data = await response.json()
    
    if (data.success) {
      set({
        isAuthenticated: true,
        user: data.user,
      })
      return true
    }
    return false
  } catch (error) {
    console.error('登录失败', error)
    return false
  }
}
```

### 2. Token 管理

建议添加 token 管理：

```typescript
interface AuthState {
  isAuthenticated: boolean
  user: { name: string; role: string } | null
  token: string | null
  login: (username: string, password: string) => Promise<boolean>
  logout: () => void
  setToken: (token: string) => void
}

// 在请求拦截器中添加 token
axios.interceptors.request.use(config => {
  const token = useAuthStore.getState().token
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### 4. 路由配置建议

```typescript
// 公开路由（不需要登录）
const publicRoutes = [
  { path: '/', element: Home },
  { path: '/about', element: About },
  { path: '/login', element: Login },
]

// 需要登录的路由
const protectedRoutes = [
  { path: '/dashboard', element: Dashboard },
  { path: '/profile', element: Profile },
]

// 需要管理员权限的路由
const adminRoutes = [
  { path: '/admin', element: Admin, requiredRole: 'admin' },
]
```

## 性能优化

### 1. 预加载

对于重要页面，可以预加载：

```typescript
import { useEffect } from 'react'

// 鼠标悬停时预加载
<Link 
  to="/dashboard"
  onMouseEnter={() => import('@/page/dashboard')}
>
  Dashboard
</Link>
```

### 2. 路由缓存

使用 React Router 的 `loader` 功能预加载数据：

```typescript
{
  path: 'dashboard',
  element: <Dashboard />,
  loader: async () => {
    const data = await fetchDashboardData()
    return data
  },
}
```
