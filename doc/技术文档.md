# Code Create Frontend 技术文档

## 项目概述

Code Create Frontend 是一个基于现代化技术栈构建的企业级前端应用框架。项目采用 React 19 + TypeScript + Vite 8 + Ant Design 6 的技术组合，并启用了 React Compiler 进行自动性能优化。

## 技术选型

### 核心框架
- **React 19.2.4** - 最新版本的 React，支持并发特性和自动批处理
- **TypeScript 6.0.2** - 提供强类型支持，提升代码质量和开发体验
- **Vite 8.0.4** - 基于 ESM 的极速构建工具，提供快速的 HMR

### UI 框架
- **Ant Design 6.3.5** - 企业级 UI 设计语言和 React 组件库
  - 提供丰富的组件库
  - 支持主题定制
  - 内置国际化支持

### 路由管理
- **React Router 7.14.1** - 声明式路由库
  - 使用 `createBrowserRouter` API
  - 支持数据加载和错误处理
  - 类型安全的路由配置

### 状态管理
- **Zustand 5.0.12** - 轻量级状态管理库
  - 简单直观的 API
  - 支持中间件（persist）
  - 与 React Compiler 兼容

### 开发工具
- **ESLint 9.39.4** - 代码质量检查工具（flat config）
- **Prettier 3.8.3** - 代码格式化工具
- **React Compiler** - 自动优化 React 组件性能

### 核心功能实现

#### 1. 路由系统

使用 React Router 7 的 `createBrowserRouter` 实现声明式路由管理：

```typescript
// src/router/index.tsx
import { createBrowserRouter } from 'react-router-dom'
import BasicLayout from '@/layouts'
import Home from '@/page/home'
import About from '@/page/about'
import Dashboard from '@/page/dashboard'

const router = createBrowserRouter([
  {
    path: '/',
    element: <BasicLayout />,
    children: [
      {
        index: true,
        element: <Home />,
      },
      {
        path: 'about',
        element: <About />,
      },
      {
        path: 'dashboard',
        element: <Dashboard />,
      },
    ],
  },
])

export default router
```

**特点：**
- 所有页面共享统一的 `BasicLayout` 布局
- 使用 `<Outlet />` 渲染子路由内容
- 支持嵌套路由和动态路由

#### 2. 主题系统

基于 Zustand 和 Ant Design 实现主题切换功能：

```typescript
// src/store/themeStore.ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

type Theme = 'light' | 'dark'

interface ThemeState {
  themeStore: Theme
  toggleTheme: () => void
  setTheme: (theme: Theme) => void
}

export const useThemeStore = create<ThemeState>()(
  persist(
    set => ({
      themeStore: 'light',
      toggleTheme: () =>
        set(state => ({
          themeStore: state.themeStore === 'light' ? 'dark' : 'light',
        })),
      setTheme: theme => set({ themeStore: theme }),
    }),
    {
      name: 'theme-storage',
    }
  )
)
```

**特点：**
- 支持亮色/暗色模式切换
- 使用 `persist` 中间件持久化到 localStorage
- 通过 Ant Design 的 `ConfigProvider` 全局配置主题

#### 3. 布局系统

`BasicLayout` 作为应用的基础布局，包含头部、内容区和底部：

```typescript
// src/layouts/index.tsx
import { Layout, ConfigProvider, theme } from 'antd'
import { Outlet } from 'react-router-dom'
import { useThemeStore } from '@/store/themeStore'

const BasicLayout = () => {
  const currentTheme = useThemeStore(state => state.themeStore)

  return (
    <ConfigProvider
      theme={{
        algorithm:
          currentTheme === 'dark'
            ? theme.darkAlgorithm
            : theme.defaultAlgorithm,
      }}
    >
      <Layout>
        <GlobalHeader />
        <Content>
          <Outlet />
        </Content>
        <GlobalFooter />
      </Layout>
    </ConfigProvider>
  )
}
```

**特点：**
- 集成主题配置
- 使用 `<Outlet />` 渲染子路由
- 统一的页面布局结构

## 技术亮点

### 1. React Compiler

项目启用了 React Compiler，通过 Babel 插件自动优化组件性能：

```typescript
// vite.config.ts
import babel from '@rolldown/plugin-babel'
import { reactCompilerPreset } from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [
    react(),
    babel({ presets: [reactCompilerPreset()] })
  ],
})
```

**优势：**
- 自动进行组件记忆化
- 减少手动使用 `useMemo`、`useCallback` 和 `React.memo`
- 提升应用性能

**注意：**
- 会影响开发和构建性能
- 需要遵循 React 规则

### 2. 路径别名

配置了 `@` 别名指向 `src` 目录，简化导入路径：

```typescript
// tsconfig.app.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}

// vite.config.ts
export default defineConfig({
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

**使用示例：**
```typescript
import Component from '@/components/Component'
import { useThemeStore } from '@/store/themeStore'
```

### 3. TypeScript 项目引用

使用 TypeScript 项目引用分离应用代码和工具配置：

- `tsconfig.json` - 根配置，定义项目引用
- `tsconfig.app.json` - 应用代码配置
- `tsconfig.node.json` - Vite 配置和 Node.js 工具配置

**优势：**
- 更快的类型检查
- 更好的代码组织
- 独立的编译选项

### 4. ESLint Flat Config

使用新的 ESLint flat config 格式：

```javascript
// eslint.config.js
import { defineConfig } from 'eslint/config'
import prettierConfig from 'eslint-config-prettier'

export default defineConfig([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      js.configs.recommended,
      tseslint.configs.recommended,
      reactHooks.configs.flat.recommended,
      reactRefresh.configs.vite,
      prettierConfig, // 避免与 Prettier 冲突
    ],
  },
])
```

## 开发规范

### 代码风格

**Prettier 配置：**
- 不使用分号
- 使用单引号
- 2 空格缩进
- ES5 风格的尾逗号
- 每行 80 字符限制

**ESLint 规则：**
- TypeScript 推荐规则
- React Hooks 规则
- React Refresh 规则
- 禁止未使用的变量和参数

### 组件开发

**推荐实践：**
1. 使用函数组件和 Hooks
2. 避免手动记忆化（React Compiler 会自动处理）
3. 使用 TypeScript 定义 Props 类型
4. 组件文件使用 `index.tsx` 命名
5. 样式文件使用 CSS Modules（`index.module.css`）

**示例：**
```typescript
// src/page/home/index.tsx
import { Card, theme } from 'antd'
import styles from './index.module.css'

function Home() {
  const { token } = theme.useToken()

  return (
    <Card className={styles.welcomeCard}>
      <h1 style={{ color: token.blue }}>欢迎使用 Code Create</h1>
    </Card>
  )
}

export default Home
```

### 状态管理

**Zustand 使用规范：**
1. 每个 store 单独文件
2. 使用 TypeScript 定义状态类型
3. 需要持久化的状态使用 `persist` 中间件
4. 避免在选择器中内联创建对象（React Compiler 兼容性）

## 性能优化

### 已实现的优化

1. **React Compiler** - 自动组件优化
2. **代码分割** - Vite 自动进行代码分割
3. **Tree Shaking** - 移除未使用的代码

## 常见问题

### Q: 如何添加新页面？

1. 在 `src/page/` 下创建新目录
2. 创建 `index.tsx` 和 `index.module.css`
3. 在 `src/router/index.tsx` 中添加路由配置

### Q: 如何切换主题？

使用 `useThemeStore` hook：
```typescript
const { toggleTheme } = useThemeStore()
toggleTheme() // 切换主题
```

### Q: 如何自定义 Ant Design 主题？

在 `BasicLayout` 的 `ConfigProvider` 中配置：
```typescript
<ConfigProvider
  theme={{
    token: {
      colorPrimary: '#1890ff',
      borderRadius: 4,
    },
  }}
>
```
